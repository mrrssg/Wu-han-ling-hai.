# 退货·追款·海外仓·哨兵

## 追款规则
- **只有豪雅(Costway)的退货可以追款，司顺(Vevor)不退不用追**
- **追款窗口 = 下单后90天内**。追款清单只列窗口内的，按剩余天数排序（≤15天标黄）；超窗的单独统计
- 在飞书「退货登记表」登记过的订单 = 已经追过款（等供应商退款），不再出现在"漏网"清单里
- 追款清单分两段：**未登记（漏网）**——退货了还没去登记的；**已登记等退款**——按账龄分档（≥150天临近180天核销线会上首页待办）
- 等待超180天 → 核销全损（written_off）

## Lowes-Autool 退货跟进四态
- **有退货跟踪号** = 买家把货退回了我们海外仓 → 不欠供应商货款，损失只有退货运费（按售价10%估），台账状态=warehouse
- 无跟踪号但「处理结果」=已回复邮件
- 弃货
- 没动静（需要跟进）

## p值（可要回比例）
p = 有跟踪号退货的货值 ÷ 全部退货货值（近90天）。它直接进定价公式的退货损失率折减——**登记的跟踪号越多，退货损失率越低，档位越松，价格越有竞争力**。所以退货务必登记跟踪号。

## 下架候选双队列（行动清单）
负期望SKU（90天退货≥2且净亏>$50）分流：
- **下架候选**：定价方案判下架的 + 价格已在档位公式价却仍亏的（修价救不了）
- **待修价验证**：价格还没修的（挂着待改价候选）或刚修完在观察期（修价<30天且修后<10单）——不会消失，观察期满仍亏自动回下架候选
- 已不在卖的自动剔除

## "下架后仍出单"哨兵
点过"已下架"或删过offer的SKU，系统每天检查是否还有新订单——有就说明平台没真正下架，首页红条+问题清单高级别报警。

## 退货数据链
平台退货接口每小时同步进数据库（mirakl_returns）；退货台账（return_case）每天早上5:30按飞书订单标记+账单重建；首页"昨日退货"直接从数据库出，不现调接口。
