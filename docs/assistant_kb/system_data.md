# 数据来源与同步时刻表

## 各数据多久更新一次
- **订单**（Macy/Lowes/BestBuy各店）：每30分钟从Mirakl同步进数据库
- **退货**（mirakl_returns）：每小时
- **交易日志**（到账/佣金/退款）：每小时
- **Offer快照**（在卖列表/价格/折扣）：每晚02:20~02:55（各店错峰）——页面上的"现价"最迟第二天凌晨更新
- **供应商价格**（豪雅/司顺）：外部feed任务同步（不在本服务器），定价全依赖它的新鲜度，超36小时会拒绝出价
- **飞书定价配置**（佣金/退货运费/尺寸）：每晚02:25~02:55
- **利润台账重建**（return_case/月账本/热力图）：每天05:30，约2.5分钟
- **Listing哨兵**：每天06:10（AI审计近2天退货SKU的listing）
- **档位评估**：每天06:30；**改价候选**：06:45；**推价回读对账**：03:30
- **零销量周推**：每周一07:30

## 首页待办的口径
全部直接读数据库，不现调平台接口。未发货=Mirakl订单状态SHIPPING（已接单待发）/WAITING_ACCEPTANCE（未接单，更急）。

## 平台接口限制
- Mirakl改offer接口（改价/删offer共用）：**1分钟1次**，一次最多50个SKU——所以批量操作永远优于连点
- 交易日志接口：20次/分、60次/时
- 每个店铺走自己固定的代理IP，互相隔离

## 已知数据盲区（如实告知）
- 132个Lowes SKU缺定价配置（92缺退货运费/7缺尺寸/33供应商表无价），暂按15%档持有、监控修不了，补齐后自动归位
- 上架天数为空的=2026年4月前的老批次（当时没记录上传日期）
- HD三店没接退货数据；Macy的not_charged残缺单口径未修
- 折扣价字段2026-07-16才开始解析，老数据可能为空（系统用原价×0.5估算）

## 模型答不了的问题
具体数字类问题（某SKU卖了多少、某月净利多少）请直接看对应页面/飞书看板——本助理不现查数据库（防止编数字），只解释规则、公式、字段和操作。
