# 分档定价方案（Lowes-Autool）

**目标**：每个运营（刘梦蝶MDLW / 明瑞瑞MRLW / 朱以超YCLW，按SKU前缀分池）的净利率都≥10%基线，同时尽量不掉销量。每天早上06:30自动重评一次全店档位。

## 三个价格档
每个在卖SKU被分到一个档位，档位决定它的定价公式除数：
- **12%档**：折扣价=(成本+退货成本预估)÷0.73 —— 退货轻的SKU，用最低价保竞争力
- **15%档**：÷0.70
- **18%档**：÷0.67 —— 退货重的SKU，多收3~6个点缓冲
（除数=1−佣金15%−档位名义毛利。名义12%实际毛利约16%，因为公式里预留的退货运费不是每单都发生、且价格向上取整）

## 怎么决定进哪一档
比两个数：
1. **需要的毛利** = 10%基线 + 该SKU的退货损失率
2. **各档公式价能给的毛利** = 逐SKU现算：(档位公式价×85% − 成本)÷档位公式价（成本、退货运费、尺寸都用SKU自己的，供应商价实时）

从12%档往上找**第一个盖得住的档**（差1个点以内算盖得住）。

**退货损失率** = (1−p可要回比例) × 窗口内退货成本货值 ÷ 窗口内销售额。
- 窗口=下单满30天的订单（30~120天前下的单），新单退货还没到齐不算
- 分子是成本货值（付给供应商的货款），不是卖价
- p = 有跟踪号退货货值占全部退货货值比例（登记退货跟踪号越多，档位越松）

**取数阶梯**（样本不够就借群体的，全带运营维度，三人不互相背锅）：
1. SKU自己：窗口内>10单，用它自己的退货记录
2. 运营×类目：该运营自己在这个类目窗口≥30单
3. 运营全池：该运营全部窗口单
4. 全店兜底

## 下架档（两条路进）
1. **退货判死**：连18%档公式价的毛利都盖不住需要毛利，且窗口≥25单坐实。样本不够25单的先按18%档卖着攒数据（"18档观察"），不判死。
2. **负期望实证**：90天退货≥2次且净亏超$50，且现价已经就是自己档位的公式价——价格是对的还在亏，修价救不了。刚修完价的有观察期（修价30天内且修后不满10单先不判）。

下架档SKU在方案页直接删offer：单删点行末按钮；批量勾行首复选框→「删除已选」。只有下架档能删，其它档提交会被拒绝。删除不可逆，删后全系统除名，并有"删后仍出单"哨兵盯着。

## 零销量SKU
- 上架<30天没出单：**观察期**，不动价
- 上架≥30天零销量：降到**12%档促活**（最低只到12%，不再往下），出单即转正常评档
- 促活改价每轮最多500个（分批看激活率），**每周一早07:30自动推送一批**，其余档位改价必须人工确认

## 价格监控联动
有档位的SKU：现毛利跌破**档位目标−2个点**才触发修价，修价目标=该档公式价。下架档SKU监控跳过（等人工删）。没档位记录的SKU走固定5%兜底线。

## 档位卡片上的数字
- **贡献业绩** = 该档SKU的90天销售额占比（选了运营/类目筛选，占比就是筛选范围内的结构）
- **退货损失率** = 该档整体的窗口退货成本÷销售额（已按可要回比例折减）
