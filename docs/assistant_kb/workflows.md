# 操作流程

## 日常改价（增量，量少）
1. 首页待办看到"待改价（Lowes-Autool）"，或直接进 价格→价格监控→待改价
2. 逐条看"为什么改"，确认没问题点推送（单推或勾选批推）
3. 推送前系统自动过五道校验（保本线/毛利自洽/原价=折扣÷0.5/供应商价新鲜度/成本突变），不过会拒推并说明原因
4. 当晚03:30自动回读Mirakl真实价对账，异常→首页红条+一键重推

## 大批量改价（Excel整体导出）
1. 价格→价格监控→整体导出，**先把右上店铺切到目标店**，点"生成Excel"
2. 系统按每个SKU的档位算价，生成后自动逐行复验（有一行错整个文件不给下载）
3. 下载文件（文件名=店铺+时间戳），可删行分批传
4. Mirakl商家后台 offers import 上传
5. 回到整体导出页点**"已上传"**——夜里同步后系统自动逐行对账，价格没生效的会红条报警
- 供应商价超过36小时没更新时会拒绝生成（数据太陈会算错价）
- 零销量促活SKU每轮只出500个，本批传完下轮自动放下一批

## 删offer（只有下架档能删）
1. 定价方案页，点下架档卡片筛出下架档
2. 单删：行末「删offer」；批量：勾行首复选框（表头全选）→「删除已选」→确认框核对清单
3. Mirakl限这类接口1分钟1次，勾10个也算1次——**删多个用勾选批量**；请求在途时全部删除按钮锁住属正常
4. 删除不可逆；删后自动全系统除名 + "删后仍出单"哨兵盯防

## 追款
1. 利润控制台→行动清单→追款清单："未登记（漏网）"里的单去飞书退货登记表登记
2. 登记过的进"已登记等退款"，按账龄催办；行内可写备注
3. 供应商退款后财务回填，次日自动核销消失

## 零销量周推（全自动，无需操作）
每周一07:30自动推500个零销量促活价（只这一类是自动的，其它改价都要人工确认）。推送记录在改价历史页。

## 报警处理（首页红条）
- **推价异常**：去待改价页看红色横幅，核对后一键重推正确价，Mirakl侧确认后手动关闭issue
- **已下架仍出单**：去店铺后台核实是否真的下架了
- **追款临近核销线**：优先催办≥150天的
- **利润率破线/负期望SKU**：看利润控制台诊断页的处方
