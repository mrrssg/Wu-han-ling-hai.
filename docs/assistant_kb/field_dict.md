# 页面字段字典

## 定价方案页（价格→定价方案）
- **SKU / 运营 / 类目**：运营按SKU前缀（MDLW=刘梦蝶、MRLW=明瑞瑞、YCLW=朱以超）
- **档位**：12%/15%/18%/下架档/零销量观察/零销量促活
- **目标毛利**：该档位的名义毛利（进定价公式的除数），实际毛利会高于它约4个点
- **90天单数 / 退货**：近90天订单数 / 退货次数
- **退货损失率**：该SKU定档用的损失率（详见公式集），红色=超5%
- **实际毛利**：实测毛利=非退货毛利÷非退货销售额（展示用，不参与档位判定）
- **数据来源**：损失率的样本层级——SKU自己(窗口>10单)/运营×类目/运营全池/全店
- **现价$**：数据库里的当前折扣价（缺折扣价时显示原价）
- **目标价$**：最新候选给它算的档位公式折扣价，红色=要涨价，蓝色=要降价，空=价格已对或没进本轮候选
- **上架天数**：从首次上架算；空=4月前的老品（没记录）
- **为什么进这一档**：档位判定的完整原因和数字（需要毛利、各档能给的毛利、样本量）
- **🟡底色**：评档输入数据存疑
- 档位卡片：**贡献业绩**=该档90天销售额占比；**退货损失率**=该档整体窗口损失率。都跟随筛选变

## 待改价清单（价格→价格监控→待改价）
- **为什么改**：候选来源（[定价方案·档位]目标毛利X% 或 监控毛利跌破线）+ 价格变化幅度
- **红色横幅**：推价异常（推了没生效/被平台改/下线）——可单个重推或全部重推
- 待改价数 = 最新定价方案候选 + 最新监控候选去重合并，减去已推成功的

## 行动清单（利润控制台→行动清单）
- **追款漏网**：退货了还没在登记表登记的（豪雅、90天窗口内）
- **已登记等退款**：按账龄五档着色，≥150天临近核销线
- **下架候选·说明列**：定价方案判下架 / 已在公式价仍亏 / 无档位记录
- **待修价验证**：负期望但价格没修的，修完观察期仍亏自动回下架候选
- **提价候选·建议%**：(10%−净利率)÷0.7，1~8%封顶

## 月度账本（利润控制台→月度影响）
- 每月两行：**预计最终赚**（待追退款按回收率记上）/ **目前实际落袋**（只认已到账）
- **毛利中预估占位**：账单没导入的部分用预估利润顶着，占比越高数字成色越低

## 常用状态值
- return_case.state：pending等退款 / recovered已退 / written_off核销 / not_charged不算退货 / warehouse退回海外仓
- 候选状态：dry_run待确认 / success已推成功 / failed失败 / skipped跳过 / alert缺数据
